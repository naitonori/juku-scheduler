## 1. このシステムでできること

「**どの講師が、どの教室で、何時から何時まで、何の授業をするか**」を、 すべての制約条件を守りながら**自動で最適に決める**システムです。

手作業で数時間かかる時間割づくりを、**ボタン1つ・数十秒**で完了できます。

### 主な特徴

| 機能 | 説明 |
|---|---|
| 教室の自動割り当て | 生徒数に合った教室を自動選択 |
| 講師の重複チェック | 同じ講師が同時刻に2つの授業を持たない |
| 裏番組NG | 最難関大・選抜講座の科目間で時間帯を分離 |
| テストゼミ対応 | テスト→解説の2フェーズを連続配置 |
| 時間固定 | 人気講座の開始時間をロック可能 |
| 待機時間の最小化 | 講師の無駄な空き時間を自動で減らす |

## 2. 環境構築（初回のみ）

### 必要なもの

- **Python 3.9 以上**
- **pip**（Python パッケージマネージャ）

### インストール手順

ターミナル（Mac）またはコマンドプロンプト（Windows）で以下を実行します：

```bash
# 1. プロジェクトフォルダへ移動
cd ~/ClaudeCode/juku-scheduler

# 2. 必要なライブラリをインストール
pip3 install ortools pandas streamlit
```

> **Windows の場合**: `pip3` を `pip` に読み替えてください。

---

## 3. 起動方法

### Web アプリを起動する

```bash
cd ~/ClaudeCode/juku-scheduler
python3 -m streamlit run app.py
```

実行すると以下のように表示されます：

```
  You can now view your Streamlit app in your browser.
  Local URL: http://localhost:8501
```

ブラウザが自動で開きます。開かない場合は **http://localhost:8501** にアクセスしてください。

### 終了する

ターミナルで `Ctrl + C` を押すと停止します。

### ターミナルだけで動かす場合（Web UI なし）

```bash
cd ~/ClaudeCode/juku-scheduler
python3 scheduler.py
```

サンプルデータで最適化を実行し、結果をターミナルに表示します。

## 4. 画面の使い方

### 4-1. サイドバー（左側）

| 項目 | 説明 |
|---|---|
| **データ入力方法** | 「CSVアップロード」か「サンプルデータで試す」を選択 |
| **ソルバー制限時間** | 最適化にかける最大秒数（5〜120秒）。授業数が多い場合は長めに |

### 4-2. サンプルデータで試す

1. サイドバーで「🧪 サンプルデータで試す」を選択（初期値）
2. 「🚀 **時間割を生成する**」ボタンをクリック
3. 数秒で結果が表示されます

> まずはこのモードで操作を覚えてください。

### 4-3. CSV アップロードで実データを使う

1. サイドバーで「📄 CSVアップロード」を選択
2. テンプレートをダウンロード（3つのボタン）
3. テンプレートに実データを入力
4. 3つの CSV を画面にアップロード
5. 「🚀 **時間割を生成する**」ボタンをクリック

## 5. CSV ファイルの作り方

CSV は **Excel で編集 → 「CSV UTF-8 (コンマ区切り)」で保存** が最も簡単です。

### 5-1. 教室 CSV（rooms.csv）

| 列名 | 必須 | 説明 | 入力例 |
|---|---|---|---|
| 校舎 | ✅ | 校舎名 | `代々木校` |
| 教室ID | ✅ | 一意な教室識別子 | `代々木校_R1` |
| 定員 | ✅ | 最大収容人数 | `30` |

**記入例：**

```csv
校舎,教室ID,定員
代々木校,代々木校_R1,8
代々木校,代々木校_R2,30
代々木校,代々木校_R3,15
長野校,長野校_R1,20
```

**注意：**
- 「教室ID」は全校舎を通して**重複しない名前**にしてください
- 命名規則は `校舎名_R連番` を推奨します

---

### 5-2. 講師 CSV（teachers.csv）

| 列名 | 必須 | 説明 | 入力例 |
|---|---|---|---|
| 講師ID | ✅ | 一意な講師識別子 | `T01` |
| 名前 | ✅ | 表示名 | `佐藤先生` |
| 稼働開始1 | ✅ | シフト開始時刻 | `09:30` |
| 稼働終了1 | ✅ | シフト終了時刻 | `21:30` |
| 稼働開始2 |   | 2つ目の時間帯（任意） | `14:00` |
| 稼働終了2 |   | 2つ目の時間帯（任意） | `18:00` |

**記入例：**

```csv
講師ID,名前,稼働開始1,稼働終了1,稼働開始2,稼働終了2
T01,佐藤先生,09:30,21:30,,
T02,鈴木先生,09:30,21:30,,
T03,高橋先生,12:30,21:30,,
```

**注意：**
- 時刻は必ず `HH:MM`（24時間表記）で入力してください
- 稼働時間帯は最大3区間まで設定可能です（稼働開始3, 稼働終了3）
- 全日稼働の場合: `09:30` 〜 `21:30`
- 午後のみの場合: `12:30` 〜 `21:30`
- 2つ目以降が不要な場合は空欄にしてください

---

### 5-3. 授業 CSV（lessons.csv）

| 列名 | 必須 | 説明 | 入力例 |
|---|---|---|---|
| 授業ID | ✅ | 一意な識別子 | `L01` |
| 講座名 | ✅ | 講座の名前 | `最難関大 英語α` |
| 校舎 | ✅ | どの校舎で実施するか | `代々木校` |
| 科目 | ✅ | 科目名 | `英語` |
| 講師ID | ✅ | 担当講師のID | `T01` |
| 授業時間(分) | ✅ | 授業の長さ（分単位） | `90` |
| 生徒数 | ✅ | 想定受講者数 | `25` |
| テストゼミ |   | テストゼミなら `はい` | `はい` |
| テスト時間(分) |   | テスト部分の長さ | `60` |
| 解説時間(分) |   | 解説部分の長さ | `60` |
| 固定開始時刻 |   | 開始時刻を固定する場合 | `13:00` |

**通常授業の記入例：**

```csv
L01,最難関大 英語α,代々木校,英語,T01,90,25,,,,
```

**テストゼミの記入例：**

```csv
L18,最難関大 英語テストゼミ,代々木校,英語,T01,0,25,はい,60,60,
```

> テストゼミの場合、「授業時間(分)」は `0` にし、代わりに「テスト時間」「解説時間」に分数を入れてください。

**時間固定の記入例：**

```csv
L19,人気講座 数学特別,代々木校,数学,T02,90,28,,,,13:00
```

**重要なルール：**

- **授業時間は 5 の倍数**にしてください（5分刻みで管理しています）
  - ✅ 45分、60分、90分、120分
  - ❌ 43分、67分
- **校舎名**は教室 CSV の校舎名と完全一致させてください
- **講師ID**は講師 CSV の講師IDと完全一致させてください
- 講座名に「**最難関大**」「**選抜**」を含めると、裏番組NGルールの対象になります

## 6. 結果の見方

### 6-1. ガントチャート

最適化完了後、校舎ごとにガントチャート（横棒グラフ）が表示されます。

- **横軸**: 時間（9:30 〜 21:30）
- **縦軸**: 教室
- **色**: 科目別（凡例が上部に表示されます）
- **マウスホバー**: 講座名・時間・講師の詳細がツールチップで表示

#### バーの読み方

| 表示 | 意味 |
|---|---|
| 色付きバー | 通常授業 or テストゼミ解説 |
| グレーのバー（🔸付き） | テストゼミのテスト時間（講師不要） |

### 6-2. 時間割一覧（テーブル）

校舎ごとのタブで切り替え可能な表形式の一覧です。

| 列 | 説明 |
|---|---|
| 講座名 | 授業の名前 |
| 種別 | 「通常授業」「テストゼミ(テスト)」「テストゼミ(解説)」 |
| 科目 | 科目名 |
| 教室 | 割り当てられた教室 |
| 開始 / 終了 | 授業の開始・終了時刻 |
| 講師 | 担当講師 |
| 生徒数 | 想定受講者数 |

### 6-3. CSV ダウンロード

「📥 結果CSVをダウンロード」ボタンで、結果を CSV ファイルとして保存できます。Excel で開いて印刷・共有が可能です。

### 6-4. ステータスの意味

| ステータス | 意味 |
|---|---|
| **OPTIMAL** | 最適な解が見つかりました（これが理想） |
| **FEASIBLE** | 解は見つかりましたが、最適ではない可能性があります |
| **INFEASIBLE** | 解が存在しません（制約が厳しすぎます） |
| **UNKNOWN** | 時間内に解が見つかりませんでした |

## 7. 最適化のルール一覧

### ハード制約（絶対に守られるルール）

| # | ルール | 説明 |
|---|---|---|
| 1 | 時間枠 | すべての授業が 9:30〜21:30 に収まる |
| 2 | 時間固定 | 固定開始時刻が指定された授業はその時刻に配置 |
| 3 | 教室重複なし | 同じ教室で同時刻に2つの授業が入らない |
| 4 | 教室定員 | 生徒数 ≦ 教室の定員 |
| 5 | 講師重複なし | 同じ講師が同時刻に2つの授業を担当しない |
| 6 | 講師シフト | 講師の稼働時間内にのみ授業を配置 |
| 7 | 裏番組NG | 最難関大/選抜の主要科目と理社が時間的に被らない |
| 8 | 講師休憩 | 異科目間で15分以上の休憩を確保 |

### 裏番組NG の詳細

「最難関大」「選抜」がつく講座に適用：

- **グループA**（国語・数学・英語）と **グループB**（物理・化学・世界史・日本史）は被ってはいけない
- **例外**: 物理と（生物 / 世界史 / 日本史 / 地理）は被ってよい
- **日本史・世界史・地理**は互いに被ってはいけない

### ソフト制約（できるだけ守るルール）

| ルール | 説明 |
|---|---|
| 講師待機時間の最小化 | 講師の最初の授業〜最後の授業の間の空き時間を最小化 |

## 8. うまくいかないときは

### 「解が見つかりませんでした」と出る場合

制約が厳しすぎてスケジュールが組めない状態です。以下を確認してください：

| チェック項目 | 対処法 |
|---|---|
| 教室の定員不足 | 生徒数を収容できる教室がその校舎にあるか確認 |
| 講師のシフト | 授業時間が講師の稼働時間内に収まるか確認 |
| 授業数が多すぎ | 教室数・時間帯に対して授業が多すぎないか確認 |
| 裏番組NG | 最難関大/選抜講座が多すぎると時間帯が足りなくなる |
| 同一講師に授業過多 | 1人の講師に授業を詰め込みすぎていないか確認 |

### 「FEASIBLE」で結果に満足できない場合

- サイドバーの**ソルバー制限時間を長く**してください（60秒〜120秒）
- 時間をかけるほど、より良い解が見つかる可能性があります

### CSV 読み込みエラーが出る場合

| エラー内容 | 原因 | 対処法 |
|---|---|---|
| KeyError: '校舎' | 列名が違う | テンプレートの列名を変えずに使う |
| 教室に存在しません | 校舎名の不一致 | 授業CSVの校舎名と教室CSVの校舎名を一致させる |
| 講師IDが見つからない | IDの不一致 | 授業CSVの講師IDと講師CSVの講師IDを一致させる |

### よくある入力ミス

- 校舎名の全角/半角の違い（`代々木校` と `代々木校 ` ← 末尾スペース）
- 時刻の形式ミス（`9:30` ではなく `09:30` を推奨）
- CSV を Excel で保存するとき「CSV UTF-8」ではなく「CSV」で保存すると文字化けする場合があります

## 9. ファイル構成

```
juku-scheduler/
├── scheduler.py          ← 最適化ソルバー本体（コアロジック）
├── app.py                ← Streamlit Web UI
├── MANUAL.md             ← このマニュアル
└── templates/
    ├── rooms.csv          ← 教室データのテンプレート
    ├── teachers.csv       ← 講師データのテンプレート
    └── lessons.csv        ← 授業データのテンプレート
```

| ファイル | 編集する？ | 説明 |
|---|---|---|
| `scheduler.py` | 通常触らない | 制約条件やルールを変更したい場合のみ |
| `app.py` | 通常触らない | 画面のレイアウトを変更したい場合のみ |
| `templates/*.csv` | ✅ 編集する | 実データに書き換えて使用 |
